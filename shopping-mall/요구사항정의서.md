# 요구사항정의서

## 1. 프로젝트 개요

### 1.1 목적

Shoppping Mall Demo의 핵심 기능을 구현한 MVP(Minimum Viable Product)를 개발하여 빠른 시장 진입과 사용자 피드백 수집을 목표로 함

### 1.2 목표

- 상품 판매를 위한 기본 기능 제공
- 사용자 친화적인 쇼핑 경험 제공
- 안정적인 주문 및 결제 프로세스 구현

### 1.3 범위

**포함 사항:**

- 회원가입/로그인
- 상품 조회 및 검색
- 장바구니
- 주문 및 결제
- 리뷰 작성

**제외 사항 (향후 버전):**

- 쿠폰/프로모션
- 포인트/적립금
- 실시간 채팅 상담
- 멀티 벤더 (다중 판매자)
- 소셜 로그인

## 2. 요구사항 명세서

### FR-001: 회원 관리

| ID        | 요구사항                    | 우선순위 |
| --------- | --------------------------- | -------- |
| FR-001-01 | 이메일 기반 회원가입 기능   | 필수     |
| FR-001-02 | 이메일/비밀번호 로그인 기능 | 필수     |

| FR-001-04 | 회원정보 조회/수정          | 필수     |
| FR-001-05 | 비밀번호 변경               | 필수     |
| FR-001-06 | 회원탈퇴                    | 필수     |

#### FR-001-01 상세 요구사항

- 이메일은 RFC 5322 기반 형식을 만족해야 하며, 공백 제거 후 검사한다.
- 이메일은 사용자 풀에서 유일해야 하며 중복 시 “이미 사용 중인 이메일입니다” 메시지로 거절한다.
- 비밀번호는 최소 8자 이상이며, 공백만 입력된 값은 허용하지 않는다.
- 필수 입력: 이메일, 비밀번호, 비밀번호 확인, 이름. 휴대폰 번호는 선택 입력이다.
- 비밀번호 불일치 시 “비밀번호가 일치하지 않습니다” 메시지와 함께 처리 실패로 응답한다.

#### FR-001-02 상세 요구사항

- 로그인은 활성 사용자만 대상으로 하며, 비활성/탈퇴 사용자는 거절한다.
- 비밀번호는 서버에 저장된 해시(bcrypt 등)와 비교해 검증한다.
- 성공 시 서버는 JWT 또는 동등한 세션 토큰을 발급하고 유효기간을 명시한다.
- 클라이언트는 로그인 성공 후 토큰과 사용자 정보를 안전한 스토리지(LocalStorage 등)에 저장하고, “아이디 저장” 옵션을 제공한다.
- 로그인 실패 시 “이메일 또는 비밀번호가 일치하지 않습니다” 메시지를 통일해 전달한다.

### 2.2 비기능 요구사항

### NFR-001: 성능

- 페이지 로딩 시간: 3초 이내
- 동시 접속자: 최소 100명 처리 가능
- 이미지 로딩 최적화 (Lazy Loading)

### NFR-002: 보안

- 비밀번호 암호화 저장 (bcrypt 또는 유사 알고리즘)
- HTTPS 통신
- SQL Injection 방어
- XSS 공격 방어
- CSRF 토큰 사용

### NFR-003: 사용성

- 반응형 웹 디자인 (모바일, 태블릿, PC)
- 직관적인 UI/UX
- 접근성 준수 (WCAG 2.1 Level A)

### NFR-004: 신뢰성

- 시스템 가용성: 99% 이상
- 데이터 백업: 일 1회
- 장애 복구 시간: 4시간 이내

### NFR-005: 유지보수성

- 코드 주석 및 문서화
- 모듈화된 아키텍처
- 버전 관리 시스템 사용 (Git)
